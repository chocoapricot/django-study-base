# ApiCache修正の実施計画

企業情報取得API（gBizINFO）と住所取得API（ZipCloud等）において、APIリクエストの結果を`ApiCache`モデルにキャッシュするように修正します。これにより、同じリクエストに対するAPI呼び出しを減らし、パフォーマンスの向上とAPI利用制限の回避を図ります。

## 修正内容

### 1. `apps/api/helpers.py` の修正
- `get_api_cache`, `set_api_cache` をインポートし、各関数内でキャッシュのチェックと保存を行うように変更します。
- キャッシュキーの形式：
    - 企業情報: `company_info_{corporate_number}`
    - 郵便番号: `zipcode_{zipcode}`
- キャッシュ有効期間：
    - パラメータ `API_CACHE_VALIDITY_PERIOD` (デフォルト: 86400秒 = 24時間) を使用します。

### 2. テストコードの追加/更新
- `apps/api/tests/test_helpers_cache.py` を作成し、キャッシュが正しく機能していることをテストします。
    - 初回リクエスト時にAPIが呼ばれ、キャッシュが保存されること。
    - 2回目以降のリクエスト時にキャッシュから取得され、APIが呼ばれないこと。

## 完了条件
- 企業情報の取得および郵便番号の取得が、初回はAPI経由で行われ、2回目以降はキャッシュから取得されること。
- 既存のテストおよび追加したテストがすべてパスすること。
